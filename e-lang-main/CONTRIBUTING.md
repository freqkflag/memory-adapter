# Contributing to E-Lang

Thank you for your interest in contributing to **E-Lang**.

E-Lang is a **formal language project**, not a feature-driven application.
Contributions must preserve precision, stability, and neurodiversity-affirming intent.

---

## Before You Contribute

Please read:
- `AGENTS.md` (required)
- `ELANG_SPEC.md`
- `ELANG.abnf`

These documents define scope and constraints.

---

## What We Welcome

### Language & Tooling
- Parsers and validators (grammar-conformant)
- Canonical formatters
- Serialization/deserialization utilities
- Test suites that lock grammar and syntax

### Documentation
- Clarifying examples
- Visual diagrams (axes, trajectories)
- Tutorials and walkthroughs
- Developer-focused explanations

### Research Support
- Figures for the manuscript
- Reference management
- Formatting for journal submission
- Replication notes

---

## What We Do NOT Accept

- New axes or reordered axes without approval
- Emotion taxonomies or label systems
- Diagnostic or therapeutic claims
- Automated affect inference
- Ethical enforcement via license changes
- Breaking changes to grammar without discussion

If in doubt, open an issue first.

---

## Contribution Workflow

1. **Open an issue** describing:
   - What you want to change
   - Why it's needed
   - Which files are affected

2. **Propose the change**
   - Additive vs modifying
   - Backward compatibility impact

3. **Implement**
   - Keep diffs minimal
   - Follow existing structure and tone

4. **Validate**
   - Grammar still parses
   - Tests pass or are updated
   - No scope drift introduced

5. **Document**
   - Update README or inline docs if behavior changes

---

## Style & Tone

- Precise, neutral, non-sensational
- Infrastructure-focused
- Respect lived experience without appropriation
- Avoid overclaiming impact

---

## Attribution

If you contribute substantial work:
- You will be credited in documentation or manuscript acknowledgements as appropriate.

---

E-Lang succeeds by staying **small, exact, and trustworthy**.
